# Personal Preferences

## TypeScript

- Never use 'any" unless 100% necessary or specifically instructed.

## Commands

- Don't run dev server commands (e.g., "bun run dev") - assume it's already running.
- Don't run build commands unless specifically told to.
- Focus on checking commands like "bun run typecheck', "bun run lint', etc.

## Package Managers

- Use bun and bunx wherever possible
- Never use npm or yarn.

## Tech Stack Preferences

When uncertain, prefer: Tailwind, TypeScript, Bun, React

## Code Style

- Always strive for concise, simple solutions.
- If a problem can be solved in a simpler way, propose it.

## Workflow

- If asked to do too much work at once, stop and state that clearly.
- **Do not commit changes unless explicitly asked to.**
- Before implementing, analyze the existing codebase, and external knowledge if needed and ask meaningful questions — don't jump straight into code.
- During building, stop and ask for clarification if you encounter issues or ambiguities.
- If a change would require significant refactoring, flag it and discuss the approach instead of silently rewriting half the codebase.

## Research & Knowledge

- Use the Context7 tool to get the latest documentation for libraries and frameworks.
- Use web fetch to find information when Context7 doesn't have what you need or for general knowledge.
